# 表达式示例

> 对应 `grammar.md` 的“3.5 表达式”和“4. 运算符优先级与结合性”。

## 1. 基本值

```lona
1
3.14
'A'
"A\x42"
"hello"
"A\x42\0"
"\xE4\xB8\xAD"
"中"
true
false
null
name
```

说明：

- `3.14` 当前默认作为浮点字面量进入语义，缺少上下文时按 `f64` 处理。
- 字符字面量 `'A'` 的值类型是 `u8`；它只接受单字节 ASCII，像 `'你'` 这样的 UTF-8 文本不进入字符语义。
- 字符与字符串都支持这些转义：`\n`、`\t`、`\r`、`\0`、`\\`、`\'`、`\"`、`\xN`、`\xNN`。
- `\xN` / `\xNN` 会写入一个原始字节；单个十六进制数字会按低 4 位处理，也就是 `\x4 == \x04`。例如 `"A\x42\x4\0"` 的字节序列就是 `41 42 04 00`。
- 字符串字面量现在按 UTF-8 byte string literal 处理；未转义的源码文本按 UTF-8 编码写入，`"hello"` 的值类型是 `u8 const[*]`。
- 由于字符串本质上是字节串，所以既可以直接写 UTF-8 文本 `"中"`，也可以写成等价的 UTF-8 字节序列 `"\xE4\xB8\xAD"`。
- 这里的 `\x...` 是原始字节写法，不是 Unicode / UTF-8 码点写法；字符串整体不做 UTF-8 合法性校验，因此也允许写入 `"\xFF"` 这类非法 UTF-8 字节。
- 编译器会为字符串字面量自动补一个结尾 `0` 字节，因此它可以直接传给 `puts(msg u8 const[*])` 这类 C 风格接口。
- `null` 是空指针字面量；它只能在指针上下文中使用，例如 `var p i32* = null`、`if p == null`。
- `&"hello"` 这类旧写法已经移除；字面量本身就是只读字节视图。
- `i32 1`、`bool(expr)` 这类 C 风格 cast 写法已经移除。

## 2. 括号表达式与括号应用

```lona
(1 + 2)
(a < b)

(
    1 +
    2
)

sum()
sum(1, 2)
sum(
    1,
    2
)
factory()(2)
```

当前 `xxx(...)` 在语义上先统一看成“括号应用”。
具体是函数调用、函数指针调用，还是未来的数组访问 / 其它重载行为，由后续语义阶段决定。
当前 grammar 显式允许在 `()` / `[]` 内部换行，也允许在赋值和二元运算符后把右侧表达式写到下一行，例如 `a +` 换行后继续写 `b`。一元运算符后面当前不允许直接换行。

## 3. 显式函数取指针

```lona
foo&<>
sum&<i32, i32>
```

这里的 `name&<...>` 表示“按形参类型列表取得顶层 `def name(...)` 的函数指针”。
当前实现里，`&<` 是一个专用起始符，因此写法应保持成连续的 `&<`。
取到的结果就是普通函数指针值，因此后续可以继续写 `cb(1)`、`make_cb()(1)` 这类间接调用。

## 4. 成员访问与链式调用

```lona
point.x
user.profile.name
obj.method(1)
math.inc(1)
pair._1
```

其中 `math.inc(1)` 这种形式表示调用 imported module `math` 暴露出来的顶层函数。
tuple 则按匿名结构体处理，字段名使用 `_1`、`_2`、`_3` 这类自动生成的成员名。
对于数值和原始字节数组，`.` 还可以命中少量被注入的内建成员，例如 `value.tobits()`、`value.tobits().toi32()`。

## 5. 元组、数组与花括号初始化

```lona
var pair <i32, bool> = (1, true)
var matrix i32[4][5] = {{}}
var grid i32[5, 4] = {}
var row i32[4] = {1, 2, 3, 4}
var row2 i32[4] = {1, 2, 3, 4,}
var row3 i32[4] = {
    1,
    2,
    3,
    4,
}
var inferred = {1, 2}
var c = Complex(real = 1, img = 2)
var d = mix(c.real, y = c.img)
var x f32 = cast[f32](1)
var raw u8[4] = x.tobits()
```

说明：

- 元组字面量现在已经支持带显式 tuple 目标类型的构造与传递，例如 `var pair <i32, bool> = (1, true)`。
- tuple 成员访问现在已经接通，例如 `pair._1`、`echo(pair)._2`。
- 固定维度数组的零初始化已经接通；`{}` 与嵌套空花括号都表示整块数组置零。
- 数组的显式元素列表已经接通；例如 `{1, 2}` 会按数组维度顺序初始化，缺失元素自动补零。
- 花括号初始化列表支持尾逗号；例如 `{1, 2,}` 和多行 `{\n    1,\n    2,\n}` 都合法。
- `var inferred = {1, 2}` 这类写法会把花括号推断成固定长度数组；这里会得到 `i32[2]`。
- 花括号层级必须匹配数组容器层级；例如 `i32[4][5]` 需要写成 `{{1}, {2}}` 这种“按行分组”的形式，而不是把所有元素拍平成一层。
- 只有花括号初始化支持这类尾逗号；普通调用、元组字面量和 `[]` 维度列表仍然不支持 `foo(1, 2,)`、`(1, 2,)`、`i32[4,]`。
- 编译器内部会先把花括号内容整理成 `initial_list` 风格的中间抽象，再根据目标数组类型完成物化；这个抽象当前不对用户开放。
- 结构体现在通过默认构造函数调用初始化，例如 `Complex(real = 1, img = 2)` 或 `Complex(1, 2)`。
- 普通函数调用和构造函数调用共用同一套命名实参语义，例如 `mix(x = 1, y = 2)`、`mix(1, y = 2)`。
- 位置实参必须写在命名实参前面；`mix(x = 1, 2)` 这种顺序会报错。
- 显式静态转换统一写成 `cast[T](expr)`；其中 `cast` 只支持内建标量和指针，不支持结构体、tuple、固定维数组等复合类型。
- `tobits()` 会返回一个按字节展开的 `u8[N]` 数组；`u8[N].toXXX()` 会把这些字节重新解释成目标数值类型。
- 整数到整数、浮点到浮点会做隐式转换；整数和浮点之间仍然要求显式 `cast[T](expr)`。

## 5.1 数组与可索引指针访问

```lona
matrix(1)(2)
grid(1, 2)
tensor(1, 2)(0)
view(1)
mixed(0)(3)
```

说明：

- 连续 `[]` 类型用多次 `()` 逐层索引。
- 同层维度组类型在一次 `()` 里写完当前层的所有索引。
- `T[*]` 可索引指针也使用同样的 `p(i)` 语法。
- 原始指针 `T*` 不能直接写 `p(i)`；如果需要受控索引，先显式转成 `T[*]`。
- 这里的 `p(i)` 是语言内建的索引访问，不等于开放 `p + i` 这类指针算术。
- 数组索引和可索引指针访问现在都直接 lower 到 LLVM GEP。

## 6. 一元运算

```lona
!flag
~mask
-value
+value
&value
*ptr
```

注意：`foo&<i32>` 这种显式函数取指针不是普通一元 `&`，而是单独的表达式形式。
另外，一元运算符当前要求和操作数写在同一行；`-`、`!`、`&`、`*` 后面不能先换行再写操作数。

## 7. 二元与赋值运算

```lona
a + b
a - b
a * b
a / b
a % b

a < b
a > b
a <= b
a >= b
a == b
a != b

a & b
a ^ b
a | b
a && b
a || b

x = 1
x += 1
x -= 1
x *= 2
x <<= 1
point.x = 1
*ptr = 42
```

说明：

- 这一轮已经把当前 grammar 里的 C 风格常见运算符接成了完整语义。
- `=` 和复合赋值当前遵循值语义：把结构体、tuple、固定维数组赋给另一个变量时，会复制值本身，不会自动形成引用别名。
- 指针和 `ref` 绑定是例外；指针赋值复制的是地址值，而 `ref` 左值写入会直接命中其绑定对象。
- `&&` 和 `||` 现在按短路语义执行，不会无条件求值右侧表达式。
- 内建运算符实现现在通过语义层的运算符解析表分派；后续要做运算符重载时，可以在这层继续扩展。
- `+` / `-` 当前不对指针开放；`T[*]` 也只能通过 `p(i)` 做索引，不支持裸偏移运算。

## 8. 优先级示例

```lona
a + b * c
(a + b) * c
obj.value + getX()
a == b + 1
a | b && c
```

按当前优先级声明，`* / %` 高于 `+ -`，移位高于比较，位运算高于逻辑运算，成员访问与括号应用也高于普通二元运算。显式函数取指针 `name&<...>` 本身作为一个单独的基础值参与后续组合。
