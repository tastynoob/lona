# 指针

> 本文只讲显式指针语义。`ref` 绑定与 `ref` 参数见 [ref.md](ref.md)，结构体方法与 `set` 规则见 [struct.md](struct.md)。

## 1. 默认仍是值语义

除显式指针外，数值、结构体、tuple 和固定维数组仍按值处理：

- `a = b` 是值拷贝赋值
- `def f(x T)` 是按值传参
- `ret x` 是按值返回

如果值里包含指针字段，拷贝时只复制指针值，不递归复制其指向对象。

## 2. 指针类型

指针类型写作：

```lona
T*
```

例如：

```lona
var p i32*
var q Point*
var r math.Point*
```

`null` 是空指针字面量。它不是单独类型，只能在明确的指针上下文中物化成 `T*` 或 `T[*]`。

## 3. 取地址与解引用

```lona
var x i32 = 1
var p i32* = &x
*p = 7
```

规则：

- `&x` 表示取得 `x` 的地址
- `*p` 表示解引用
- 指针赋值复制的是地址值，不是其指向对象
- `var p i32* = null`、`if p == null` 这类写法当前合法

自动解引用只开放给 dot-like 和 call-like 语义：

- `p.x`
- `p.method(...)`
- `p()`

这些位置会隐式补一层 `*`。
但 `&p`、`p = ...`、`p + 1` 这类位置不会自动解引用。

这条规则只补一层，而且不会叠加在显式 `*` 上：

- `(*pp).x` 只表示先显式解引用一次，再做成员访问
- 它不会继续隐式变成 `(**pp).x`

## 4. 可索引指针

除了原始指针 `T*`，当前还支持：

```lona
T[*]
```

它表示“允许索引的指针”，不是未定长数组值。

例如：

```lona
var raw u8[4] = {1, 2, 3, 4}
var p u8* = &raw(0)
var view u8[*] = p
view(1) = 9
```

规则：

- `T*` 是原始指针，不允许 `p(i)`，也不允许 `p + i`
- `T[*]` 允许 `p(i)` 这种受控索引
- `T*` 和 `T[*]` 当前允许隐式转换，但要求元素类型 `T` 完全一致
- `T[*]` 不携带长度信息，因此它不是 slice/span

## 5. 可取地址对象

`&expr` 只允许作用于可寻址左值。

当前可取地址的对象包括：

- 普通变量
- 结构体字段
- 固定维数组元素
- `*ptr`

当前不可直接取地址的对象包括：

- 字面量
- 算术表达式
- 临时构造值
- 普通函数返回值
- 其它右值

## 6. 数组与指针

`lona` 不采用 C 风格的数组自动退化。

例如：

```lona
var arr i32[4] = {1, 2, 3, 4}
var p i32[4]* = &arr
```

这里：

- `arr` 的类型是 `i32[4]`
- `&arr` 的类型是 `i32[4]*`

因此下面这种写法不成立：

```lona
var arr i32[4] = {1, 2, 3, 4}
var p i32* = arr
```

如果需要数组地址，必须显式写 `&arr`。

## 7. 三种常见写法的区别

```lona
i32[4]*   // 数组指针
i32[*]    // 可索引指针
i32*[4]   // 指针数组
```

含义分别是：

- `i32[4]*`：指向一个 `i32[4]` 值的指针
- `i32[*]`：一个可索引的 `i32` 指针
- `i32*[4]`：一个长度为 4 的数组，数组元素类型是 `i32*`

## 8. 当前不支持的能力

当前文档不把下列能力定义成稳定特性：

- 指针算术
- 完整的指针比较规则
- `T[]` 的稳定用户语义
