# 表达式示例

> 对应 `docs/grammer.md` 的“3.5 表达式”和“4. 运算符优先级与结合性”。

## 1. 基本值

```lona
1
3.14
"hello"
true
false
name
```

说明：

- `3.14` 当前默认作为浮点字面量进入语义，缺少上下文时按 `f64` 处理。
- 字符串字面量目前仍然只保留占位；未接通的部分会给出明确诊断。
- `i32 1`、`bool(expr)` 这类 C 风格 cast 写法已经移除。

## 2. 括号表达式与括号应用

```lona
(1 + 2)
(a < b)

sum()
sum(1, 2)
factory()(2)
```

当前 `xxx(...)` 在语义上先统一看成“括号应用”。
具体是函数调用、函数指针调用，还是未来的数组访问 / 其它重载行为，由后续语义阶段决定。

## 3. 显式函数取指针

```lona
foo&<>
sum&<i32, i32>
```

这里的 `name&<...>` 表示“按形参类型列表取得顶层 `def name(...)` 的函数指针”。
当前实现里，`&<` 是一个专用起始符，因此写法应保持成连续的 `&<`。
取到的结果就是普通函数指针值，因此后续可以继续写 `cb(1)`、`make_cb()(1)` 这类间接调用。

## 4. 成员访问与链式调用

```lona
point.x
user.profile.name
obj.method(1)
math.inc(1)
pair._1
```

其中 `math.inc(1)` 这种形式表示调用 imported module `math` 暴露出来的顶层函数。
tuple 则按匿名结构体处理，字段名使用 `_1`、`_2`、`_3` 这类自动生成的成员名。
对于数值类型，`.` 还可以命中被注入的内建成员函数，例如 `1.tof32()`、`1.5.toi32()`。
原始位模式则通过 `tobits()` 暴露为 `u8[N]`，例如 `value.tobits()`、`value.tobits().toi32()`。

## 5. 元组与数组初始化

```lona
var pair <i32, bool> = (1, true)
var matrix i32[4][5] = {{}}
var grid i32[5, 4] = {}
var x f32 = 1.tof32()
var raw u8[4] = x.tobits()
```

说明：

- 元组字面量现在已经支持带显式 tuple 目标类型的构造与传递，例如 `var pair <i32, bool> = (1, true)`。
- tuple 成员访问现在已经接通，例如 `pair._1`、`echo(pair)._2`。
- 固定维度数组的零初始化已经接通；`{}` 与嵌套空花括号都表示整块数组置零。
- 显式元素列表初始化仍未实现；像 `{1, 2}`、`{{1}}` 这类写法会给出明确诊断。
- 数值显式转换使用注入的成员函数形式，例如 `1.tof32()`、`x.toi64()`。
- `tobits()` 会返回一个按字节展开的 `u8[N]` 数组；`u8[N].toXXX()` 会把这些字节重新解释成目标数值类型。
- 整数到整数、浮点到浮点会做隐式转换；整数和浮点之间仍然要求显式 `.toXXX()`。

## 5.1 数组访问

```lona
matrix(1)(2)
grid(1, 2)
tensor(1, 2)(0)
```

说明：

- 连续 `[]` 类型用多次 `()` 逐层索引。
- 同层维度组类型在一次 `()` 里写完当前层的所有索引。
- 数组索引现在已经直接 lower 到 LLVM GEP。

## 6. 一元运算

```lona
!flag
~mask
-value
+value
&value
*ptr
```

注意：`foo&<i32>` 这种显式函数取指针不是普通一元 `&`，而是单独的表达式形式。

## 7. 二元与赋值运算

```lona
a + b
a - b
a * b
a / b
a % b

a < b
a > b
a <= b
a >= b
a == b
a != b

a & b
a ^ b
a | b
a && b
a || b

x = 1
x += 1
x -= 1
x *= 2
x <<= 1
point.x = 1
*ptr = 42
```

说明：

- 这一轮已经把当前 grammar 里的 C 风格常见运算符接成了完整语义。
- `&&` 和 `||` 现在按短路语义执行，不会无条件求值右侧表达式。
- 内建运算符实现现在通过语义层的运算符解析表分派；后续要做运算符重载时，可以在这层继续扩展。

## 8. 优先级示例

```lona
a + b * c
(a + b) * c
obj.value + getX()
a == b + 1
a | b && c
```

按当前优先级声明，`* / %` 高于 `+ -`，移位高于比较，位运算高于逻辑运算，成员访问与括号应用也高于普通二元运算。显式函数取指针 `name&<...>` 本身作为一个单独的基础值参与后续组合。
